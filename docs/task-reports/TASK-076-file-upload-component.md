# T076: FileUpload 컴포넌트

## 무엇을 만들었나요?

파일을 업로드할 수 있는 React 컴포넌트를 만들었어요. 마치 이메일에 파일을 첨부할 때처럼, 사용자가 코드 파일을 쉽게 업로드할 수 있게 해주는 기능이에요.

## 왜 이렇게 만들었나요?

사용자가 코드 학습을 위해 파일을 올릴 때, 두 가지 방법을 지원하고 싶었어요:
1. **드래그 앤 드롭**: 파일을 마우스로 끌어다 놓는 방식
2. **파일 선택 버튼**: 클릭해서 파일을 선택하는 전통적인 방식

또한 잘못된 파일(너무 크거나, 지원하지 않는 형식)을 미리 걸러내서 사용자 경험을 좋게 만들었어요.

## 어떻게 작동하나요?

1. 사용자가 파일을 드래그하면, 드롭 영역이 시각적으로 변해요 (테두리 색 변경)
2. 파일을 놓으면, 설정된 규칙(파일 크기, 허용 확장자)에 따라 검증해요
3. 검증을 통과하면 부모 컴포넌트에 파일 목록을 전달해요
4. 검증에 실패하면 에러 메시지를 보여줘요

## TDD로 어떻게 테스트했나요?

RED-GREEN 사이클을 따랐어요:
- **RED**: 먼저 15개의 테스트를 작성 (파일 선택, 드래그 앤 드롭, 크기 검증, 비활성화 상태 등)
- **GREEN**: 테스트를 통과시키는 최소한의 코드 작성
- 모든 테스트가 통과하는지 확인

## 관련 파일

- `frontend/src/components/upload/FileUpload.tsx` - 컴포넌트 구현
- `frontend/src/components/upload/FileUpload.test.tsx` - 테스트 코드

## 배운 개념

- **HTML5 File API**: 브라우저에서 파일을 다루는 방법
- **Drag and Drop Events**: dragEnter, dragLeave, dragOver, drop 이벤트
- **React useRef와 useCallback**: 성능 최적화를 위한 훅 사용

## 주의사항

- jsdom 환경에서는 `DataTransfer` API가 완벽하게 지원되지 않아서, 테스트에서 mock이 필요해요
- 파일 크기 검증은 클라이언트 측에서만 이루어지므로, 서버 측에서도 반드시 검증해야 해요
