# T082: ProjectDetail 태스크 통합

## 무엇을 만들었나요?

기존 프로젝트 상세 페이지에 태스크 관련 기능을 추가했어요. 이제 프로젝트에서 태스크 목록을 보고, 새 태스크를 만들 수 있어요.

## 왜 이렇게 만들었나요?

프로젝트와 태스크는 밀접하게 연결되어 있어요:
- 하나의 프로젝트에 여러 태스크가 포함될 수 있어요
- 태스크는 프로젝트 컨텍스트 안에서 관리되어야 해요
- 사용자가 프로젝트를 열면 바로 태스크 목록을 볼 수 있어야 해요

## 어떻게 작동하나요?

1. 프로젝트 상세 페이지 진입 시 태스크 목록을 자동으로 불러와요
2. 통계 카드에 태스크 개수가 표시돼요
3. "새 태스크" 버튼으로 CreateTaskModal을 열 수 있어요
4. 각 태스크는 TaskCard로 표시되고, 클릭하면 상세 페이지로 이동해요
5. 태스크 생성 후 목록이 자동으로 새로고침돼요

## TDD로 어떻게 테스트했나요?

- 12개의 테스트 케이스 작성
- projectService와 taskService를 mock해서 테스트
- 태스크 목록 표시, 빈 상태, 태스크 생성, 네비게이션 등 테스트

## 관련 파일

- `frontend/src/pages/ProjectDetail.tsx` - 페이지 수정
- `frontend/src/pages/ProjectDetail.test.tsx` - 테스트 코드

## 배운 개념

- **TanStack Query의 invalidateQueries**: 데이터 변경 후 캐시 새로고침
- **컴포넌트 통합**: 기존 페이지에 새 기능을 자연스럽게 추가하기
- **조건부 렌더링**: 로딩, 빈 상태, 데이터 있는 상태를 구분해서 표시

## 주의사항

- 태스크 생성 성공 시 토스트 메시지와 함께 목록이 새로고침돼요
- 서버 오류 시 에러 토스트가 표시돼요
- 지원 언어 목록은 상수로 정의되어 있어요 (나중에 백엔드에서 받아올 수도 있음)
