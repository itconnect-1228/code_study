# T081: CreateTaskModal 컴포넌트

## 무엇을 만들었나요?

새 태스크를 만드는 모달 다이얼로그를 만들었어요. 세 가지 업로드 방법(파일, 폴더, 붙여넣기)을 탭으로 전환하면서 사용할 수 있어요.

## 왜 이렇게 만들었나요?

하나의 모달에서 세 가지 업로드 방법을 모두 제공하면:
- 사용자가 상황에 맞는 방법을 선택할 수 있어요
- 화면 전환 없이 빠르게 태스크를 생성할 수 있어요
- UI가 일관되고 직관적이에요

## 어떻게 작동하나요?

1. "새 태스크" 버튼을 클릭하면 모달이 열려요
2. 태스크 제목을 입력해요
3. 탭을 선택해서 업로드 방법을 고르세요:
   - **파일 탭**: 파일 드래그 앤 드롭 또는 선택
   - **폴더 탭**: 폴더 선택
   - **붙여넣기 탭**: 코드 직접 입력 + 언어 선택
4. "태스크 만들기" 버튼을 클릭하면 생성돼요

## TDD로 어떻게 테스트했나요?

- 23개의 테스트 케이스 작성
- 렌더링, 탭 전환, 각 업로드 방법별 동작, 제출, 에러 처리 등 테스트
- 기존 FileUpload, FolderUpload, PasteCode 컴포넌트를 조합해서 테스트

## 배운 개념

- **Tabs 컴포넌트**: Radix UI 기반의 탭 UI
- **컴포넌트 합성**: 기존 컴포넌트들을 조합해서 새 기능 만들기
- **폼 상태 관리**: 여러 입력의 상태를 조합해서 유효성 검증

## 주의사항

- 붙여넣기 방식은 "코드 추가" 버튼을 먼저 클릭해야 코드가 저장돼요
- 폼 제출 시 제목이 비어있으면 버튼이 비활성화돼요
- 에러 발생 시 에러 메시지가 표시되고 모달은 닫히지 않아요
